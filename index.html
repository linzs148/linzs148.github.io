core Java Chapter3

Posted on 2020-06-29

《Java核心技术 卷一》第三章笔记

关于markdown中字体/图片的更改语法

联机API文档

主干知识点：

float的有效位数为6-7位，大约±3.402 823 47E+38F

double的有效位数为15位，大约±1.797 693 134 862 315 70E+308

特殊的浮点数：Float/Double.POSITIVE_INFINITY，Double.NEGATIVE_INFINITY，Double.NaN

用Double.isNaN()方法检查一个浮点数是否为NaN

从Java10开始，对于局部变量，可以不明确变量的类型，而是用var关键字接收赋值过来的对象，该关键字自动检测赋值操作中的对象类型

#Java中没有无符号整数类型，必要时需要将byte→short→int→long类型转换以扩大数据的范围（Byte.toUnsignedShort、Short.toUnsignedInt、Integer.toUnsignedLong）

用enum枚举类型限定变量的取值在给定的范围之内，从而保证了得到变量的格式的统一

#整数被0除会产生一个异常，而浮点数被0除则会得到无穷大或NaN

Java中没有幂运算，需要借助Math.pow()方法

R：import java.lang.Math.*；

如果对于计算结果的精度有比较高的要求可以使用StrictMath类进行运算

Math.round()方法对浮点数进行四舍五入操作，结果返回一个long类型的整数，必要时需要强制转换为int类型

算术右移（补0/1）>> 逻辑右移（补0）>>>

String.join()方法实现字符串的拼接，访问器方法

R：对于字符串的检查 if ( str != null && str.length() != 0 ) #顺序一定是先检查是否为null后检查是否为空串，否则直接在null上调用length()方法会报错

使用System.out.printf()方法进行格式化输出，标志和转换符参考这里

String.format()方法可以创建一个格式化的字符串而不打印输出

用Scanner进行文件读取，例如：Sacnner in = new Scanner(Path.of(“xxx.txt”), ScandardCharsets.UTF_8)；

（这里的Path.of构造一个Path对象，类似于Path.get）

用PrintWriter进行文件读取，例如：PrintWriter out = new PrintWriter(“xxx.txt”, ScandardCharsets.UTF_8)；

可以用shell的重定向语法将文件关联到System.in和System.out：java MyProg <in.txt> out.txt

Java提供带标签的break能够跳出多重循环（P75）， 类似于goto

BigInteger类和BigDecimal类能够满足任意长度数字序列的数值，需要注意的是这两个类加减乘除符号没有被重载，需要调用对应的方法进行运算

用Arrays.toString()方法可以把数组转换为String便于打印输出（多维数组使用Arrays.deepToString()）

数组的赋值为浅拷贝，指向同一块区域，一处修改所有引用都会变动

深拷贝要使用Arrays.copyOf(oldArray, length)，去除某一部分可以试用Arrays.copyOfRange(oldArray, start, end)

Arrays.binarySearch()实现二分查找

Arrays.sort()使用优化过的快排对数组进行排序

Math.random()返回[0, 1)之间随机浮点数

P89页多维数组在内存中的结构

细节知识点：

如果main方法正常退出，那么Java应用程序的退出码为0，如果希望终止程序时返回其他的退出码，就需要使用System.out方法

Java中整数的范围与运行代码的机器无关，奠定了Java能够跨平台运行的基础

如果对于计算结果的精度有比较高的要求可以使用StrictMath类进行运算

简单地输入使用java.util.Scanner即可，但这种输入方式效率太低，不适合OJ时使用（OJ时建议使用BufferedReader）

StringBuilder类效率较高，但不支持多线程方式修改，StringBuffer类与之相反

用F12键进入开发者模式，然后鼠标右键点击左上角的刷新图标选择第三个清空缓存并硬性重新加载就能够清理缓存加载出更新后的内容（如果一次不行就多试几次，，，）
